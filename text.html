What is VCS ?

To give you more material for a better insight into this system, I will provide you with an explanation about how the so-called Version Control System works.
Version Control System - VCS will manage and store all the files of the entire project and also the modifying history of each file.
With each modification, a version will be created .
The system will help developers to clone and store different versions of source code from the repository,
each change of source code from local devices will be committed and posted to the server where the repository is located .

What is Git ?

it is a Distributed Version Control System - DVCS, arguably, it is one of the most commonly used systems today.
Git provides each developer with a separated repository, containing a complete history of changes.

How does Git work ?

Basically, version control systems like CVS, Subversion, Perforce, Bazaar, etc. all share a common behavior, which is to view the information they store as a collection of files and their changes over time.
To be clear, eventhough the latest version has no modification, VCS still store it as the latest version.
However, Git does not work according to the above principles, instead, Git saves its information as snapshots.
Every time you delegate, Git creates a snapshot with a reference to it. For maximum efficiency, Git will not archive if the file is committed but without any changes.
